얼굴인식 sdk 성능 분석

  • 웹툰에 '마주쳤다' 라는 웹툰이 한 때 인기를 끌었는데 들어보셨을지 모르겠습니다.
  • 웹툰내에선 얼굴인식을 해서 얼굴을 웹툰화 해주는 씬이 있습니다.
  • sdk 라이센스 비용이 증가하면서, 후보 sdk와의 성능을 비교하는 과제를 맡게 되었습니다.

문제 접근

  • 첫 과제라 긴장도 되고, 성능을 비교하려면 무엇부터 해야할지 고민이 되었습니다.
  • sdk 부분만 바꿔 테스트를 해보았는데, 그것만으로 정확히 어떤게 더 좋다 가늠하기 어려웠습니다.
  • 자세한 내부 로직은 업무 내용이라 말씀드릴 순 없지만,
    • 핵심은 캡처된 얼굴의 눈/코/입 등 주요 부분을 crop한 결과를 서버에 보낸다는 것입니다.
    • 결국 얼마나 눈/코/입을 정확히 crop 하냐가 sdk의 성능을 가늠할 수 있는 주요한 포인트였습니다.

다만 문제는,

  • 코드가 sin, cos, tan 등 수학적인 용어들이 가득하여 정확히 어떻게 crop을 하는지를 보기가 어려웠습니다.
  • 디버깅을 해가며, 반환된 bitmap이 올바르게 나왔는지를 볼 순 있었으나 직접 비교하는데 한계가 있었습니다.

첫 과제인데 기대를 저버릴 순 없습니다!
여러가지 다양한 테스트를 해보았고, 가장 비교가 쉽도록 샘플 앱을 만들게 되었습니다.

결론적으론, 샘플앱 제작이 sdk 성능을 파악하는데 큰 도움이 되었습니다.

  • sdk에서 추출한 스켈레톤을 카메라 위에 그렸고, crop된 영역을 바로 실시간으로 볼 수 있게 하였습니다.
    • 이전 대학교 종설 때 해본 과제를 여기서 활용할 기회가 있어서 신기했습니다.
  • 로직으로 구한 bitmap도 바로 출력할 수 있게 하고, 버튼만 누르면 sdk가 교체되도록 하였습니다.
  • 샘플앱이 있으니 어떤 sdk가 성능이 좋은지 직관적으로 알 수 있었고, 더불어 기존 crop 로직에 이슈가 있는 것도 발견하여 개선하였습니다.

큰 작업은 아니었지만, 데모앱으로 팀내 의사결정에 대한 시간을 세이브할 수 있어서 뿌듯했습니다.
